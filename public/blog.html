Discuss the scope of var, let, and const

var:

  • Function-scoped or globally scoped.
  • Hoisted to the top of their scope.
  • Can be re-declared and updated.
  • Not block-scoped; can be accessed outside of block statements.

let:

  • Block-scoped (limited to the block {} where it's defined).
  • Hoisted to the top of their block scope but not initialized until the declaration is evaluated.
  • Cannot be re-declared in the same scope.
  • Supports reassignment; the value can be updated after declaration.

const:

  • Block-scoped (limited to the block {} where it's defined).
  • Must be initialized with a value upon declaration and cannot be reassigned to a different value.
  • For objects and arrays, properties or elements can be modified, but the variable reference cannot be reassigned.
Tell us the use cases of null and undefined

Undefined:

  • Default absence: Used when a variable has been declared but not assigned any value.
  • Function returns: Returned implicitly by functions that do not explicitly return a value.
  • Property absence: When trying to access a non-existent property in an object.
  • Variable not found: Accessing a variable that doesn't exist.

Null:

  • Intentional absence: Represents the intentional absence of any object value.
  • Clearing/resetting: Used to explicitly reset a value or indicate no current object reference.
  • Signifying absence: Often used to show that a variable has no meaningful value or object assigned to it.
What do you mean by REST API?

REST (Representational State Transfer) is an architectural style for creating networked applications. RESTful APIs, following REST principles, focus on:

  • Statelessness: Requests must contain all necessary information, and the server doesn't store client context between requests.
  • Resource-Based: Resources, identified by URIs, are the core of the API. Examples include users, posts, etc.
  • Uniform Interface: Uses standard HTTP methods (GET, POST, PUT, DELETE) to manipulate resources.
  • Representation: Resources are represented in formats like JSON or XML for client-server interaction.
  • Stateless Communication: Clients and servers interact in a stateless manner.

RESTful APIs are popular due to their simplicity, scalability, and compatibility with the web. They facilitate effective communication between different systems over the internet by following interoperable principles.
